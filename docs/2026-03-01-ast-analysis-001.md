# AST Module Code Review

**Date:** 2026-03-01
**Commit:** 8f84bf9 (`feat: add typed AST layer with zero-cost newtype wrappers over CST`)
**Scope:** `crates/internal/brink-syntax/src/ast/` (mod.rs, nodes.rs, support.rs) + lib.rs changes

## Overall

Clean, well-structured typed AST layer following the standard rowan pattern (same approach as rust-analyzer). Zero-cost newtype wrappers over CST nodes, a macro for boilerplate, support helpers for traversal. Every node kind in `SyntaxKind` has a corresponding `ast_node!` declaration — nothing missing. Tests cover the main accessor paths.

## Findings and resolutions

### 1. `Tag::text()` silent fallback on missing `#`

**Finding:** `strip_prefix('#').unwrap_or(trimmed)` silently returns the raw text if the `#` is missing. The parser guarantees a `HASH` token via `bump_assert(HASH)`, so the fallback is dead code.

**Resolution:** Switched to token-based extraction — skip the `HASH` token, collect remaining token text, trim. Consistent with the rowan idiom of walking structured tokens rather than string-manipulating concatenated text.

### 2. `StringLit::raw_text()` silent fallback on missing quotes

**Finding:** Same pattern as #1. Opening quote is guaranteed (parser enters on `QUOTE`), but closing quote may be absent — unterminated strings emit an error and close the node without a trailing `QUOTE`.

**Resolution:** Added comments documenting the invariants. Left the implementation as-is because the error-recovery case is real, and the node's interior is complex (string interpolation with inline logic) making token-based extraction non-trivial for minimal benefit.

### 3. `ReturnStmt::has_value()` fragile detection logic

**Finding:** Used a dual-branch check (token scan OR child node presence). The token scan was dead code — the parser always wraps return values in expression nodes. The token branch only added false-positive risk from error recovery.

**Resolution:** Added `value() -> Option<Expr>` accessor, derived `has_value()` from it. Removed the fragile token scan entirely.

### 4. Untyped `SyntaxNode` leaking through typed layer

**Finding:** `InfixExpr::lhs()`, `rhs()`, `Assignment::target()`, `DivertTargetExpr::target()`, `ListExpr::items()` all returned raw `SyntaxNode`, leaking the untyped CST through a typed API.

**Resolution:** Added a full `Expr` enum with 12 variants covering every expression node kind the Pratt parser produces. Updated `InfixExpr::lhs/rhs` and `Assignment::target` to return `Option<Expr>`. Changed `DivertTargetExpr::target` to `Option<Path>` and `ListExpr::items` to `Iterator<Item = Path>` (these are unambiguously `PATH` nodes).

### 5. `AuthorWarning::text()` dual `TODO:` / `TODO` stripping

**Finding:** Stripped `TODO:` then fell back to bare `TODO`. Both branches are reachable (parser doesn't enforce the colon), but operating on concatenated text is inconsistent with the structured tree.

**Resolution:** Switched to token-based extraction — `skip_while` past `KW_TODO`, optional `COLON`, and leading trivia, `take_while` until `NEWLINE`. Consistent with finding #1.

### 6. `ChoiceBullets::is_sticky()` doesn't reject mixed bullets

**Finding:** `is_sticky()` returned `true` if *any* `PLUS` token existed. The parser accepts mixed bullets (`*+`, `+*`). The reference ink compiler uses the first bullet to determine stickiness.

**Resolution:** Changed `is_sticky()` to check only the first `STAR`/`PLUS` token (matching reference behavior). Added `is_mixed() -> bool` for diagnostic passes to detect degenerate mixed-bullet input.

### 7. No `Display` impl on AST nodes

**Finding:** No `Display` implementation. `Debug` delegates to syntax node debug output.

**Resolution:** Added `Display` impl to the `ast_node!` macro, delegating to `syntax.text()`. Also implemented `Display` on the `Expr` enum.

### 8. Missing accessors for content-bearing nodes

**Finding:** About a third of node types were declared via `ast_node!` but had no `impl` block — empty shells with no accessors.

**Resolution:**
- Added `Expr`-returning accessors to `ChoiceCondition::expr()`, `InnerExpression::expr()`, `ParenExpr::inner()`.
- Defined `content_node_accessors!` macro for shared content-element methods (`texts`, `inline_logics`, `glue_nodes`, `escapes`). Applied to `ChoiceStartContent`, `ChoiceBracketContent`, `ChoiceInnerContent`, `BranchContent`.
- Added `BranchContent::divert()` for divert access.
- Added full accessors to `MultilineBranchBody` (content elements + `logic_lines`, `divert`, `content_lines`).
- Trivial leaf nodes (`Text`, `Escape`, `GlueNode`, `EmptyLine`, `StrayClosingBrace`, `SequenceSymbolAnnotation`, `SequenceWordAnnotation`) left without accessors — `.syntax().text()` is sufficient.
